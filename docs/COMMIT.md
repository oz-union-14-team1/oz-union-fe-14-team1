# 📝 Commit Convention

> [Conventional Commits](https://www.conventionalcommits.org/) 기반의 커밋 컨벤션을 따릅니다.

---

## 커밋 메시지 형식

```
<타입>: <커밋 제목>(#이슈 번호)

- 무엇을 변경했는지 또는 왜 변경했는지 설명
```

---

## 커밋 타입

| 타입       | 설명                     | 예시                           |
| ---------- | ------------------------ | ------------------------------ |
| `feat`     | 새로운 기능 추가         | `feat: 위시리스트 구현(#12)`   |
| `fix`      | 버그 수정                | `fix: 버그 수정(#12)`          |
| `refactor` | 코드 리팩토링            | `refactor: 코드 리팩토링(#12)` |
| `docs`     | 문서 수정                | `docs: README 수정(#12)`       |
| `style`    | 코드 포맷팅, 세미콜론 등 | `style: 코드 포맷 수정(#12)`   |
| `build`    | 빌드 관련 수정           | `build: 빌드 설정 수정(#12)`   |
| `chore`    | 빌드, 패키지 매니저 설정 | `chore: 초기 세팅(#12)`        |
| `perf`     | 성능 개선                | `perf: 렌더링 최적화(#12)`     |
| `ci`       | CI 설정                  | `ci: GitHub Actions 설정(#12)` |
| `test`     | 테스트 코드 추가 및 수정 | `test: 유닛 테스트 추가(#12)`  |

---

## 커밋 메시지 작성 예시

### 단일 변경사항

```
feat: 로그인 기능 구현(#12)
```

### 상세 설명이 필요한 경우

```
feat: 다크 모드 구현(#17)

- 시스템 설정에 따라 초기 테마를 설정하는 기능 추가
- 사용자가 테마를 수동으로 변경할 수 있는 토글 버튼 구현
```

---

## 주의사항

> - 커밋 제목은 **50자 이내**, **소문자**로 작성
> - 제목과 본문 사이에 **빈 줄** 추가
> - 본문에는 **무엇을**, **왜** 변경했는지 작성
> - 이슈 번호 **필수** 포함 `(#이슈번호)`
